## 1. 행렬 곱셈 (Matrix Multiplication)

### 수식

$$C_{ij} = \sum_{k=1}^{K} A_{ik} \cdot B_{kj}$$

### 직관

`C[i][j]` = **A의 i번째 행**과 **B의 j번째 열**이 얼마나 "닮았는가" (내적).

값이 클수록 두 벡터가 같은 방향을 가리킨다.

### Shape 규칙

```
A: (m × k)
B: (k × n)
      ↑↑
      이 두 숫자가 반드시 같아야 함
C = A @ B → (m × n)
```

### AI에서 쓰이는 곳

```python
y = x @ W + b    # 신경망 Dense 레이어
Q @ K.T          # Transformer Attention score
```

### 코드

```python
C = A @ B                          # 실전
C = np.einsum('ik,kj->ij', A, B)   # Transformer 코드에서 자주 등장
```

## 2. 내적 (Dot Product)

### 수식

$$\mathbf{a} \cdot \mathbf{b} = \sum_{i=1}^{n} a_i b_i = \|\mathbf{a}\| \|\mathbf{b}\| \cos\theta$$

### 직관

두 벡터가 가리키는 방향이 얼마나 같은가.

- 같은 방향 → 큰 양수
- 직교(90°) → 0
- 반대 방향 → 큰 음수

행렬 곱셈의 각 칸이 결국 내적 하나.

## 3. 코사인 유사도 (Cosine Similarity)

### 수식

$$\cos(\mathbf{a}, \mathbf{b}) = \frac{\mathbf{a} \cdot \mathbf{b}}{\|\mathbf{a}\| \cdot \|\mathbf{b}\|}$$

### 직관

내적을 두 벡터의 크기로 나눠서 **방향만** 비교.
벡터가 길든 짧든 방향이 같으면 1이 나온다.

| 관계             | 값   |
| ---------------- | ---- |
| 완전히 같은 방향 | 1.0  |
| 직교             | 0.0  |
| 반대 방향        | -1.0 |

### AI에서 쓰이는 곳

- **단어 임베딩 유사도:** "왕"과 "황제"의 벡터가 얼마나 비슷한가
- **Retrieval:** 질문 벡터와 문서 벡터 중 가장 가까운 것 찾기
- **Attention:** 쿼리와 키가 얼마나 관련 있는가

### N×N 행렬 버전

```python
norms  = np.linalg.norm(X, axis=1, keepdims=True)  # (N,1)
X_norm = X / norms                                  # 정규화 — 브로드캐스팅
sim    = X_norm @ X_norm.T                          # (N,N) 한 번에
```

## 4. 브로드캐스팅 (Broadcasting)

### 규칙

오른쪽부터 차원을 비교:

1. **같으면** → OK
2. **하나가 1이면** → 1인 쪽이 상대방 크기로 늘어남
3. **둘 다 1도 아니고 다르면** → 에러
4. **차원 수가 부족하면** → 앞에 1을 채워서 비교

### 핵심 패턴

```python
# 패턴 1: 행 방향 확장 (편향 더하기)
(32, 64) + (64,)     →  (32, 64)
# b가 (1,64)로 취급되어 32번 복사됨 (개념적으로)

# 패턴 2: 열 방향 확장 (샘플별 가중치)
(32, 64) + (32, 1)   →  (32, 64)
# c가 열 방향으로 64번 복사됨

# 패턴 3: Attention 마스크 — 핵심!
(B, H, T, T) + (T, T)  →  (B, H, T, T)
# mask가 (1,1,T,T)로 취급되어 모든 배치·헤드에 적용
```

### AI에서 쓰이는 곳

```python
y = x @ W + b           # b: (out,) → 브로드캐스팅
scores = scores / d_k   # 스칼라 브로드캐스팅
scores = scores + mask  # (B,H,T,T) + (T,T) — Transformer
x = x * gamma + beta    # Layer Normalization
```

## 5. 자주 나오는 NumPy / PyTorch 패턴

| 코드             | 의미                            | 브로드캐스팅 목적   |
| ---------------- | ------------------------------- | ------------------- |
| `x[None, :]`     | 앞에 차원 1 추가 `(N,) → (1,N)` | 행 방향 확장 준비   |
| `x[:, None]`     | 뒤에 차원 1 추가 `(N,) → (N,1)` | 열 방향 확장 준비   |
| `x.unsqueeze(0)` | PyTorch 버전의 `x[None]`        | 배치 차원 추가      |
| `keepdims=True`  | 연산 후에도 차원 유지           | 브로드캐스팅 호환   |
| `axis=-1`        | 마지막 차원 방향으로 연산       | shape 독립적인 코드 |
